## 二、知识点详解

### 2.1 机器学习三要素（模型、策略、算法）

李航《统计学习方法》中的经典框架：

| 要素 | 定义 | 面试要点 |
|------|------|----------|
| **模型** | 假设空间，即输入到输出的映射集合 | 区分参数模型（线性回归）与非参数模型（KNN）；说说模型容量与过拟合的关系 |
| **策略** | 选择最优模型的准则，即损失函数 + 正则化项 | 经验风险最小化 vs 结构风险最小化；怎么加正则化项 |
| **算法** | 求解最优模型的具体计算方法 | 凸优化 vs 非凸优化；SGD / Adam 的收敛性保证 |

**2025年新认知**：在大模型时代，"模型"的层次更加丰富——预训练基底 + 适配器(LoRA) + 提示模板，三者共同构成实际部署的模型。

**面试常问**："举例说明模型、策略、算法三者如何体现在GPT的训练中？"

### 2.2 三大学习范式

#### 监督学习（Supervised Learning）
- 输入-输出对 (x, y)，目标是学习映射 f: X -> Y
- 典型任务：分类、回归、序列标注
- 大模型应用：SFT（监督微调）

#### 无监督学习（Unsupervised Learning）
- 只有输入 x，没有标签 y，目标是发现数据内在结构
- 典型任务：聚类（K-Means, DBSCAN）、降维（PCA, t-SNE）、密度估计
- 大模型应用：预训练（Causal LM本质上是无监督/自监督学习）、对比学习

#### 强化学习（Reinforcement Learning）
- Agent 在环境中行动，获得奖励信号，目标是最大化累计奖励
- 核心：状态 S、动作 A、奖励 R、策略 π、价值函数 V/Q
- 大模型应用：RLHF（PPO算法）、DPO、GRPO（DeepSeek-R1使用的）

**2025年趋势**：RLHF -> DPO -> GRPO 的演进路径是高频考点。需要理解为什么DeepSeek-R1放弃了RLHF中的奖励模型。

### 2.3 损失函数

损失函数是**策略**的核心组成部分，面试必问。

#### 交叉熵损失（Cross-Entropy Loss）

分类任务中最常用的损失函数：

```
L = -Σ y_i * log(p_i)
```

其中 y_i 是真实标签（one-hot或概率分布），p_i 是模型预测的概率分布。

- **二分类特例**（Binary Cross-Entropy / Log Loss）：
  ```
  L = -[y * log(p) + (1-y) * log(1-p)]
  ```

- **2025年常见用法**：LLM训练中交叉熵只对"输出token"计算，不对输入token计算损失（这是很多人忽略的细节）

#### 均方误差（MSE / L2 Loss）

回归任务标准损失：
```
L = (y - ŷ)²
```

**面试注意**：MSE配合Sigmoid在分类任务中的梯度消失问题（交叉熵的梯度更陡，收敛更快）。

#### KL散度（Kullback-Leibler Divergence）

衡量两个概率分布之间差异的非对称度量：
```
KL(P || Q) = Σ P(i) * log(P(i) / Q(i))
```

- KL(Q || P) ≠ KL(P || Q) — 不对称性
- 在蒸馏中，教师模型输出分布 Q，学生模型 P，使用 KL(P || Q) 或反向KL取决于实际需求
- **2025热点**：DPO中隐式地最小化了偏好数据的KL散度

#### Hinge Loss

SVM中使用，也用于某些对比学习场景：
```
L = max(0, 1 - y * ŷ)
```
其中 y ∈ {-1, 1}。

#### 对比损失（Contrastive Loss / InfoNCE）

SimCLR、CLIP等对比学习模型的核心损失：
```
L = -log( exp(sim(q, k⁺)/τ) / Σ exp(sim(q, k)/τ) )
```

**2025年**：CLIP的对比损失在多模态大模型中仍被广泛使用。InfoNCE损失成为连接文本-图像的核心技术。

### 2.4 优化器

#### SGD（随机梯度下降）

最基本优化器：
```
θ = θ - η * ∇L(θ)
```
- 问题：震荡严重、收敛慢、对学习率敏感
- 改进：Momentum（动量法）

#### Adam（Adaptive Moment Estimation）

2015年提出，至今仍是主力优化器之一：
```
m = β₁ * m + (1-β₁) * g    # 一阶动量（均值）
v = β₂ * v + (1-β₂) * g²   # 二阶动量（方差）
θ = θ - η * m / (√v + ε)
```
- 优点：自适应学习率、对超参数不敏感、适合非平稳目标
- 缺点：在Transformer训练中可能不收敛（已有论文证明）

#### AdamW（Adam with Decoupled Weight Decay）

2019年提出，当前LLM训练的标准优化器：

**关键改进**：将权重衰减（Weight Decay）从L2正则化中分离出来，不再直接在梯度中加 λ*θ，而是单独在参数更新时做衰减：

```
# AdamW 更新逻辑（简化伪代码）
m = β₁ * m + (1-β₁) * g
v = β₂ * v + (1-β₂) * g²
θ = θ - η * m / (√v + ε) - η * λ * θ   # 注意：最后一项是独立的权重衰减
```

面试中**高频问题**：为什么LLM训练用AdamW而不是Adam？
答：Adam的L2正则化会与自适应学习率产生耦合，导致正则化效果与学习率相关。AdamW解耦后效果显著更好，特别是在GPT-2/BERT/Llama等Transformer模型中。

#### LAMB（Layer-wise Adaptive Moments）

Google 2019年提出，用于大规模Batch训练（如BERT在64K batch size下训练）：
```
# 核心思想：逐层调整学习率
r = ||θ|| / ||m / (√v + ε)||     # 信任比率（trust ratio）
θ = θ - η * r * m / (√v + ε)     # 用r调整每层的学习率
```

- 在超大Batch（64K+）训练中稳定收敛
- 在LLaMA等模型的大规模预训练中仍有使用

#### 2025年优化器新进展

- **Sophia**：使用随机Hessian对角估计的优化器，比AdamW快2倍
- **LOMO (Low-Memory Optimizer)**：减少优化器状态的显存占用，适合全参数微调
- **GaLA** (2025)：梯度对齐优化，在MoE训练中表现突出

### 2.5 过拟合与正则化

#### L1 / L2 正则化

- **L2 (Ridge)**：损失函数加 λ * ||θ||²。权重趋向于小但不为零。
- **L1 (Lasso)**：损失函数加 λ * |θ|。权重趋向于稀疏（部分变为零）。

**面试重点**：L1为什么会产生稀疏解？答：L1正则项在0点不可导，梯度下降中某次更新后权重可以精确归零。从贝叶斯角度看，L1对应Laplace先验，L2对应Gaussian先验。

#### Dropout

训练时随机丢弃部分神经元（比例为 p）：

```python
# PyTorch 实现方式
dropout = nn.Dropout(p=0.1)  # 保留 90% 的神经元
output = dropout(x)           # 训练时随机丢弃，推理时缩放
```

- **2025更新**：大模型中 Dropout 使用极少。GPT-3/Llama 等模型在预训练中通常不使用Dropout（因为数据量和模型容量足够大，过拟合不严重）。但在小规模 SFT 数据中，Dropout 仍有使用。

#### Batch Normalization (BN)

对每个batch在 channel 维度做归一化：
```
ŷ = (x - μ_batch) / √(σ²_batch + ε)
y = γ * ŷ + β
```

- 优点：加速收敛、缓解梯度消失、有一定正则化效果
- 缺点：对batch size敏感（小batch效果差）；训练/推理行为不一致（推理时用全局统计量）

#### Layer Normalization (LN)

对每个样本在 feature 维度做归一化：
```
ŷ = (x - μ_sample) / √(σ²_sample + ε)
y = γ * ŷ + β
```

- **为什么Transformer用LN而不是BN**：NLP任务中不同样本的序列长度不同，BN的batch统计不稳定。LN对每个样本独立归一化，对batch size不敏感。
- **Pre-LN vs Post-LN**：现代LLM几乎全部使用Pre-LN（残差之前做LN），训练更稳定。

#### RMSNorm (Root Mean Square Layer Normalization)

LLaMA系列使用的简化版LN：
```
ŷ = x / √(mean(x²) + ε) * γ
```

- 去掉了LN的均值中心化步骤，计算更高效
- LLaMA 1/2/3、Mistral、Qwen 等都使用 RMSNorm

### 2.6 激活函数

#### ReLU 系列

**ReLU（Rectified Linear Unit）**：
```
f(x) = max(0, x)
```
- 优点：计算简单、缓解梯度消失（正区间梯度为1）、稀疏激活
- 缺点：Dead ReLU问题（负区间梯度为0，神经元可能永久死亡）

**Leaky ReLU**：
```
f(x) = max(αx, x), α 通常为 0.01
```
- 解决Dead ReLU问题，负区间有小的梯度

**PReLU (Parametric ReLU)**：
- 与Leaky ReLU类似，但 α 是可学习的参数

#### GELU（Gaussian Error Linear Unit）

2016年提出，2018年BERT使用后成为NLP主流激活函数：
```
GELU(x) = x * Φ(x)
```
其中 Φ(x) 是标准正态分布的CDF。

近似计算（常用）：
```
GELU(x) ≈ 0.5 * x * (1 + tanh(√(2/π) * (x + 0.044715 * x³)))
```

**为什么GELU比ReLU好**：GELU是ReLU的平滑版本，且带有概率解释（输入的门控机制），在Transformer中表现更优。

#### Swish / SiLU

Google 2017年提出，定义：
```
Swish(x) = x * σ(βx)
```
其中 σ 是 sigmoid 函数。当 β=1 时称为 SiLU（Sigmoid Linear Unit）。

#### SwiGLU（Swish-Gated Linear Unit）

**2025年最热门的激活函数**，PaLM、LLaMA、Mistral、Qwen、DeepSeek全线使用。

```
SwiGLU(x, W, V, W_out) = (Swish(xW) ⊙ xV) * W_out
```

其中 ⊙ 是element-wise乘法。本质是GLU（门控线性单元）的一个变体，门控使用Swish激活。

**为什么LLM都用SwiGLU**：
1. 门控机制让网络有更强的表达能力
2. Swish平滑非零梯度帮助训练稳定
3. 相比ReLU，在语言建模任务上稳定提升约0.5-1.0 perplexity

**注意**：SwiGLU的FFN层需要三个权重矩阵（W, V, W_out），参数量比标准FFN多33%。因此一些模型（如LLaMA）使用"down_proj, gate_proj, up_proj"的命名。

### 2.7 注意力机制

注意力机制的核心思想：**让模型关注输入中最重要的部分**。

#### Scaled Dot-Product Attention

公式（Attention Is All You Need, 2017）：

```
Attention(Q, K, V) = softmax(QKᵀ / √d_k) * V
```

其中 Q（Query）、K（Key）、V（Value）都是从输入通过线性投影得到的矩阵。除以 √d_k 防止 softmax 输出陷入极端值（梯度消失）。

```python
# PyTorch 实现
def scaled_dot_product_attention(Q, K, V, mask=None):
    d_k = Q.size(-1)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))
    attn_weights = F.softmax(scores, dim=-1)
    return torch.matmul(attn_weights, V), attn_weights
```

**面试常问**：为什么除以 √d_k？
答：当 d_k 很大时，QKᵀ 的点积值方差约为 d_k，导致 softmax 的输入落在梯度极小区域。除以 √d_k 可以让方差回到1，使得 softmax 的梯度更健康。

**Flash Attention（2022-2025演化）**：不计算完整的注意力矩阵，而是通过分块（tiling）在SRAM中逐块计算，大幅减少HBM访问。2025年FlashAttention-3已支持FP8计算。

#### Multi-Head Attention (MHA)

将Q, K, V分成h个头，每个头独立计算注意力，最后拼接：

```
MultiHead(Q, K, V) = Concat(head₁, ..., head_h) * W_O
head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
```

- h 通常在 8-96 之间（GPT-4有96头？）
- 不同头可以关注输入的不同子空间

**2025年新变体**：
- **GQA (Grouped Query Attention)**：多个query头共享一组key/value头。LLaMA 2/3、Mistral使用。查效率高、推理速度快。
- **MQA (Multi-Query Attention)**：所有query头共享一组key/value。推理最快但质量略降。

#### Self-Attention

Q, K, V 来自同一个序列，计算序列内部各位置之间的相关性。

应用：Transformer Encoder中的每个token间。

**因果（Causal）Self-Attention**：GPT等自回归模型使用，只允许关注当前位置及之前的位置（通过上三角mask）。

```python
# 因果掩码
causal_mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1).bool()
# 注意力分数在mask=True的位置被设为 -inf
```

#### Cross-Attention

Q 来自一个序列（如解码器），K和V来自另一个序列（如编码器输出）。

应用：Transformer Decoder中，解码器查询编码器的输出；T5/BART中的Cross-Attention层；多模态模型中文本查询图像特征。

### 2.8 Transformer架构详解

#### 整体架构

```
Transformer = Encoder + Decoder  (原始论文)
            = Encoder only       (BERT, RoBERTa)
            = Decoder only       (GPT系列, LLaMA, Qwen)
            = Encoder-Decoder    (T5, BART)
```

**2025年现状**：几乎所有主流LLM都是 **Decoder-only** 架构（GPT风格）。

#### Encoder（编码器）

结构：Embedding + N × Transformer Block

每个Block：
1. Multi-Head Self-Attention（双向注意力）
2. Add & LayerNorm (或 Pre-LN: LayerNorm -> Attention -> Residual)
3. Feed-Forward Network（FFN）
4. Add & LayerNorm

#### Decoder（解码器）

结构：Embedding + N × Transformer Block

每个Block：
1. Masked Multi-Head Self-Attention（因果注意力）
2. Add & LayerNorm
3. Cross-Attention（Q来自decoder, K/V来自encoder）
4. Add & LayerNorm
5. Feed-Forward Network
6. Add & LayerNorm

#### Positional Encoding（位置编码）

Transformer没有递归结构，需要显式注入位置信息。

**原始方法（Sinusoidal PE）**：
```
PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```
- 优点：可以外推到训练时未见过的位置
- 缺点：表达能力有限

**可学习位置编码**（BERT、GPT-2）：
- 将位置编码作为可学习的Embedding
- 缺点：不能外推，需要设定最大长度

**RoPE（Rotary Position Embedding）**（2021-2025年标准）：
- 将位置编码通过旋转矩阵注入到Q和K中
- 相对位置编码，天然支持外推
- LLaMA、Mistral、Qwen、DeepSeek全部使用
- 详见02-文档的"长上下文技术"部分

#### Feed-Forward Network（前馈网络）

标准FFN：
```
FFN(x) = W₂ * ReLU(W₁ * x + b₁) + b₂
```

**2025年标准FFN**（SwiGLU）：
```
FFN_Gated(x) = W_out * (SiLU(x * W_gate) ⊙ x * W_up)
```

参数量约为标准FFN的1.33倍，但效果更好。

#### Residual Connection（残差连接）

每个子层输出加回输入：
```
output = LayerNorm(x + Sublayer(x))
```

**Pre-LN**（当前标准）：
```
output = x + Sublayer(LayerNorm(x))
```

Pre-LN比Post-LN训练更稳定，允许更大的学习率，不必使用学习率预热到很高（但仍然需要warmup）。

#### 2025年Transformer改进

- **LLaMA架构**（Meta）：RMSNorm + RoPE + SwiGLU + GQA + Pre-LN
- **DeepSeek-V2**：Multi-head Latent Attention（MLA，压缩KV Cache） + DeepSeekMoE
- **Gemma架构**（Google）：GeGLU激活 + Post-Norm（Google风格）
- **混合Transformer+SSM**：如Jamba（AI21 Labs）在部分层使用Mamba代替Attention

### 2.9 词嵌入（Word Embedding）

#### Word2Vec（2013）

- **CBOW**：根据上下文词预测中心词
- **Skip-gram**：根据中心词预测上下文词
- 训练技巧：Negative Sampling, Hierarchical Softmax
- 局限：静态嵌入，无法处理一词多义

#### GloVe（Global Vectors, 2014）

基于词共现矩阵的分解：
```
J = Σ f(X_ij) * (w_iᵀ * w̃_j + b_i + b̃_j - log X_ij)²
```
- 利用全局统计信息，训练效率高
- 语义类比：king - man + woman ≈ queen

#### BERT Embedding（2018）

Token Embedding + Segment Embedding + Position Embedding

- BERT的嵌入层包含：WordPiece子词单元（约30K词汇表）、位置编码（可学习）、段落编码
- BERT的每层输出都可以作为"特征嵌入"，一般来说-1层（最后一层）或-2层效果最好

#### Token Embedding（现代大模型）

**Byte-Pair Encoding (BPE)**：
- 从字符开始，迭代合并最频繁的相邻token对
- GPT-2/3/4、LLaMA、Qwen 使用

**SentencePiece / Unigram**：
- 基于概率语言模型的子词切分
- T5、LLaMA（部分版本）、Gemma 使用

**tiktoken（OpenAI 2023-2025）**：
- OpenAI官方的快速BPE Tokenizer
- cl100k_base（GPT-4, GPT-4-turbo使用，100K词汇表）
- o200k_base（GPT-4o使用，200K词汇表，更多多语言支持）

**2025年新进展**：
- **字节级Tokenization**（如Mamba）：直接处理字节，没有固定的词汇表
- **多模态Tokenizer**：将图像patch映射到LLM的嵌入空间（如Qwen-VL的视觉编码器）
- **超长Tokenizer**：GPT-4o的200K词汇表支持更高效的编码

### 2.10 评估指标

#### 分类指标

| 指标 | 公式 | 适用场景 |
|------|------|----------|
| Accuracy | (TP+TN)/(TP+TN+FP+FN) | 平衡数据集 |
| Precision | TP/(TP+FP) | 关注"假阳性"风险（如垃圾邮件） |
| Recall | TP/(TP+FN) | 关注"假阴性"风险（如疾病检测） |
| F1 Score | 2*P*R/(P+R) | 类别不平衡 |

**Macro vs Micro vs Weighted F1**：
- Macro：每类算F1再平均（每类平等）
- Micro：全局混淆矩阵算F1（每个样本平等）
- Weighted：按每类样本数加权平均

#### 文本生成指标

**Perplexity（困惑度）**：
```
PPL = exp(-1/N * Σ log P(w_i | context))
```
- 衡量语言模型对下一个token预测的不确定性
- 越低越好（理论最小值是1）
- **注意**：不同Tokenizer下PPL不可比较；PPL低不代表生成质量好

**BLEU（Bilingual Evaluation Understudy）**：
- 基于n-gram精确率（通常n=1-4）
- 加上短句惩罚（Brevity Penalty）
- 主要用于机器翻译评估

**ROUGE（Recall-Oriented Understudy for Gisting Evaluation）**：
- ROUGE-N：计算n-gram召回率
- ROUGE-L：基于最长公共子序列
- 主要用于文本摘要评估

#### 2025年评估新趋势

- **LLM-as-a-Judge**：用GPT-4/Claude等大模型评估生成质量。MT-Bench、AlpacaEval、Chatbot Arena
- **多维度评估**：事实性、安全性、有帮助性、完整性
- **Human Evaluation**：仍然是最可靠的评估方式（Chatbot Arena的Elo排名）

### 2.11 面试常考数学

#### 概率论基础

**条件概率与贝叶斯定理**：
```
P(A|B) = P(B|A) * P(A) / P(B)
```

面试应用：朴素贝叶斯分类器、贝叶斯正则化、贝叶斯在RLHF中的应用。

**最大似然估计（MLE）**：
```
θ_MLE = argmax_θ P(D|θ)
```

等价于最小化负对数似然（NLL），与交叉熵损失的等价关系：

```math
L = -log P(y|x) = Cross-Entropy(y, ŷ)
```

**最大后验估计（MAP）**：
```
θ_MAP = argmax_θ P(θ|D) = argmax_θ P(D|θ) * P(θ)
```

L1正则化对应Laplace先验的MAP，L2正则化对应Gaussian先验的MAP。

#### 信息论

**熵（Entropy）**：
```
H(X) = -Σ P(x) * log P(x)
```
表示随机变量X的不确定性。

**交叉熵（Cross-Entropy）**：
```
H(P, Q) = -Σ P(x) * log Q(x)
```
衡量真实分布P与预测分布Q之间的差异。

**KL散度**：
```
D_KL(P || Q) = H(P, Q) - H(P) = Σ P(x) * log(P(x)/Q(x))
```

**互信息（Mutual Information）**：
```
I(X; Y) = H(X) - H(X|Y) = H(Y) - H(Y|X)
```
衡量X和Y共享的信息量。

**面试问题**：为什么KL散度不对称？什么时候用正向KL（P||Q），什么时候用反向KL（Q||P）？
- 正向KL（P||Q）要求Q覆盖所有P不为0的区域（zero-avoiding），导致Q趋向于"平均"所有模式
- 反向KL（Q||P）鼓励Q在P最大的区域集中（zero-forcing），导致Q趋向于"选择"一个模式
- 在蒸馏中，教师->学生通常使用反向KL

#### 线性代数重点

**矩阵分解**：
- **SVD（奇异值分解）**：A = UΣVᵀ。应用：推荐系统、PCA、低秩近似
- **低秩分解**：A ≈ UV，U∈R^{m×r}, V∈R^{r×n}。应用：LoRA微调（两个低秩矩阵的乘积）

**特征值与特征向量**：
```
Av = λv
```
应用：PCA（协方差矩阵的特征向量）、图的谱聚类、收敛性分析

**范数**：
- L0：非零元素个数（NP难，通常用L1近似）
- L1：绝对值之和 |x|₁ = Σ|x_i|（稀疏诱导）
- L2：欧几里得范数 |x|₂ = √(Σx_i²)（权重衰减）
- L∞：最大绝对值 |x|_∞ = max|x_i|

**向量空间**：
- 内积与相似度（cosine similarity）：cos(A,B) = A·B / (||A|| * ||B||)
- Softmax的几何解释：将向量映射到概率单纯形

## 三、面试常见问题（3-5个）

### Q1: 为什么现代LLM都使用LayerNorm而不是BatchNorm？Pre-LN和Post-LN有什么区别？

**回答要点**：
1. NLP任务中序列长度不固定，BN需要沿batch维度统计，不稳定
2. LN对每个样本独立归一化，与batch size无关
3. Post-LN在残差连接后做归一化，训练时梯度容易爆炸/消失，需要学习率warmup
4. Pre-LN在每个子层输入前做归一化，梯度更稳定，训练更顺畅，当前所有主流模型使用

### Q2: 解释AdamW相比Adam的关键改进，以及为什么LLM训练要用它？

**回答要点**：
1. Adam的权重衰减（L2正则化）融合在梯度更新中，与自适应学习率耦合
2. AdamW将权重衰减单独解耦：θ = θ - lr * adam_update - lr * weight_decay * θ
3. 解耦后正则化效果与学习率无关，对Transformer训练至关重要
4. 实验证明AdamW在BERT/GPT等模型上收敛更快、泛化更好

### Q3: 为什么SwiGLU在2024-2025年成为LLM的标准激活函数？它和ReLU比有什么优劣势？

**回答要点**：
1. SwiGLU = Swish(W_gate * x) ⊙ (W_up * x)，通过门控机制增强表达能力
2. 相比ReLU：SwiGLU平滑且非单调，梯度信息更丰富
3. 在语言模型任务上稳定提升0.5-1.0 PPL
4. 代价：参数量增加33%（需要三个投影矩阵），计算量更大
5. 现在几乎所有主流模型（LLaMA, Mistral, Qwen, DeepSeek）都默认使用

### Q4: 详细推导KL散度在DPO/RLAIF中的应用，为什么它是不对称的？

**回答要点**：
1. DPO损失中隐式使用KL散度约束策略偏离：
   L_DPO(π_θ, π_ref) = -E[log σ(β * log(π_θ(y_w|x)/π_ref(y_w|x) - π_θ(y_l|x)/π_ref(y_l|x)))]
2. 正向KL(P||Q)对Q在P非零区域的覆盖很严格（mode-covering）
3. 反向KL(Q||P)让Q集中在P的某个模式上（mode-seeking）
4. DPO使用的forward KL更适合偏好优化

### Q5: 解释Flash Attention的原理，为什么能加速Transformer训练？

**回答要点**：
1. 标准Attention计算完整N×N矩阵（O(N²)显存），瓶颈在HBM带宽
2. Flash Attention通过分块（tiling）在SRAM中逐块计算
3. 避免写回HBM完整的attention矩阵，减少显存读写
4. 使用重新计算（recomputation）在前向中不保存attention矩阵，反向时重新计算
5. FlashAttention-3 (2025) 支持FP8，进一步加速
