    数据结构

    来源：CyC2018/CS-Notes 算法章节 + Krahets/hello-algo 图解数据结构 考情：央国企笔试权重最高的科目，考查范围以基础概念和算法性质为主，侧重对数据结构核心性质的理解与辨析。

    一、线性表

    线性表是最基本的数据结构，由 n（n ≥ 0）个相同类型数据元素组成的有限序列。逻辑上相邻的元素在物理存储上可能相邻（顺序存储）也可能不相邻（链式存储）。

    1. 数组（Array）

    数组是一组连续内存空间中存储的相同类型数据元素的集合，是最简单、最基础的数据结构。

    核心性质：

    • 随机访问：通过下标直接计算地址，表达式为 base + index × 元素大小，时间复杂度为 O(1)。这是数组最显著的优势。
    • 插入与删除：需要移动后续元素以保持连续性，平均时间复杂度为 O(n)。插入位置越靠前，需要移动的元素越多。
    • 空间连续性：数据在内存中连续排列，有利于利用 CPU 缓存的空間局部性原理，遍历性能好。

    动态数组（如 Java 的 ArrayList、C++ 的 vector）：

    • 底层仍使用固定数组，当元素数量达到容量上限时触发扩容。
    • 扩容策略通常是新容量为原容量的 1.5 倍（Java ArrayList）或 2 倍（C++ vector）。
    • 每次扩容涉及数据拷贝（将原数组内容复制到新分配的内存空间），但通过均摊分析，N 次插入操作的总时间复杂度仍为 O(N)，即均摊 O(1)。
    • 扩容是以空间换时间的典型设计。

    多维数组的存储方式：

    • 行优先（C/C++、Python）：按行依次存储，a[i][j] 的地址 = 基地址 + (i × 列数 + j) × 元素大小。遍历时按行访问能充分利用缓存。
    • 列优先（Fortran、MATLAB）：按列依次存储，a[i][j] 的地址 = 基地址 + (j × 行数 + i) × 元素大小。适合按列访问的场景。

    数组的局限：大小固定（动态数组也不能无限制增长，扩容有开销），插入和删除操作低效。

    2. 链表（Linked List）

    链表通过指针将一组节点串联起来，每个节点包含数据域和指针域。节点在内存中不连续存储，依靠指针维持逻辑顺序。

    三种基本类型：

    类型 节点结构 特点
    单向链表 data + next（指向下一节点） 只能沿一个方向遍历，尾节点的 next 为空
    双向链表 prev + data + next（各指向前驱和后继） 可双向遍历，删除操作更灵活，但每个节点多一个指针开销
    循环链表 尾节点的 next 指向头节点 可从任意节点出发遍历整个链表，常用于循环队列

    基本操作复杂度：

    • 查找：O(n)（需要从头遍历）
    • 插入与删除（已知目标节点的位置）：O(1)（只需修改指针）
    • 插入与删除（未知位置，需先查找）：O(n)

    典型操作：

    • 链表反转：可通过迭代法（维护三个指针 prev、curr、next 依次反转指向）或递归法实现。
    • 快慢指针：用两个指针以不同速度遍历链表。典型应用包括检测环（Floyd 判环算法，快指针每次走两步、慢指针每次走一步，若相遇则有环）、找链表中点（快指针到末尾时慢指针正好在中点）、找倒数第 K 个节点（快指针先走 K 步，然后快慢指针同步移动）。
    • 合并两个有序链表：通过比较两个链表当前节点的值，取较小者接入结果链表。

    数组与链表的对比：

    对比维度 数组 链表
    访问方式 随机访问 O(1) 顺序访问 O(n)
    插入/删除（已知位置） O(n) O(1)
    空间利用率 连续存储，无额外指针开销 每个节点额外存储指针
    缓存友好性 高（空间局部性好） 低（节点分散）
    大小调整 动态数组扩容有拷贝成本 天然动态增长

    适用场景：频繁随机访问选择数组；频繁插入删除且不常随机访问选择链表。

    3. 栈（Stack）

    栈是后进先出（LIFO, Last In First Out）的线性结构，所有操作限制在栈顶一端进行。

    实现方式：

    • 顺序栈（数组实现）：用数组存储元素，栈顶指针指向当前栈顶位置。需要处理栈满时的扩容问题（可参考动态数组的扩容策略）。
    • 链栈（链表实现）：用头插法链表实现，每次入栈将新节点插入链表头部，出栈移除头部节点。链栈不存在栈满限制。

    基本操作：push（入栈，在栈顶插入元素）、pop（出栈，移除并返回栈顶元素）、peek（查看栈顶元素但不移除）、isEmpty（判断栈是否为空）。所有基本操作时间复杂度均为 O(1)。

    典型应用场景：

    • 函数调用栈：程序运行时，每次函数调用将返回地址、局部变量等压入调用栈；函数返回时出栈恢复执行现场。
    • 表达式求值：将中缀表达式转换为后缀表达式（逆波兰式），需要操作数栈和运算符栈配合。计算后缀表达式时只需一个操作数栈。
    • 括号匹配：遍历表达式时，左括号入栈；遇到右括号时出栈匹配，若匹配失败则括号不配对。
    • 浏览器的前进后退：用两个栈分别存储后退历史记录和前进历史记录。

    4. 队列（Queue）

    队列是先进先出（FIFO, First In First Out）的线性结构，在一端（队尾）插入，在另一端（队头）删除。

    普通队列：可用数组或链表实现。链表实现时需要维护队头（first）和队尾（last）两个指针。

    循环队列（数组实现）：

    • 为了解决数组队列的"假溢出"问题（队尾已满但队头有空位），将数组在逻辑上视为环形。
    • 判空条件：front == rear（队头指针等于队尾指针）
    • 判满条件：(rear + 1) % capacity == front（牺牲一个存储单元来区分空和满）
    • 元素个数计算：(rear - front + capacity) % capacity
    • 另一种判满方式：维护一个 size 变量记录当前元素个数，可避免牺牲存储单元。

    双端队列（Deque, Double-Ended Queue）：

    • 两端均可进行插入和删除操作，兼具栈和队列的功能。
    • Java 的 ArrayDeque、Python 的 collections.deque 均为典型实现。
    • 应用场景：滑动窗口最大值、回文判断等。

    优先队列（Priority Queue）：

    • 按优先级而不是插入顺序出队，优先级最高的元素最先出队。
    • 底层通常用堆实现（大根堆或小根堆）。
    • 基本操作的时间复杂度：插入 O(log n)，删除 O(log n)，查看优先级最高元素 O(1)。
    • 应用场景：任务调度（高优先级任务先执行）、Dijkstra 最短路径算法、Huffman 编码。

    栈与队列的相互实现：

    • 两个栈实现队列：一个栈负责入队（push 直接压入该栈），另一个栈负责出队（出队时若出队栈为空，则将入队栈所有元素依次弹出并压入出队栈）。均摊复杂度 O(1)。
    • 两个队列实现栈：入栈时向非空队列插入元素；出栈时将当前队列中除最后一个元素外的所有元素移到另一个队列，然后移除最后一个元素。

    二、树与二叉树

    树是一种非线性数据结构，以分层的方式组织数据。二叉树是每个节点最多有两个子树的树结构。

    1. 二叉树基础概念

    重要二叉树类型：

    | 类型 | 定义 | 特点 | |:—-:|:—-:|::—-:| | 满二叉树 | 除叶子节点外每个节点都有两个子节点，且所有叶子在同一层 | 节点数为 2^h - 1（h 为层数） | | 完全二叉树 | 除最底层外其余层均满，最底层节点尽可能靠左排列 | 可用数组高效存储，适合堆的实现 | | 二叉搜索树（BST） | 左子树所有节点值 < 根节点值 < 右子树所有节点值 | 中序遍历为递增序列 | | 平衡二叉树（AVL） | 任一节点的左右子树高度差的绝对值不超过 1 | 严格平衡，保证查找效率 |

    二叉树的重要性质：

    • 第 i 层最多有 2^(i-1) 个节点（根节点为第 1 层）。
    • 深度为 k 的二叉树最多有 2^k - 1 个节点。
    • 叶子节点数 = 度为 2 的节点数 + 1（度为 2 表示有两个子节点）。
    • 完全二叉树中，节点 i（从 1 开始编号）的左子节点为 2i，右子节点为 2i+1，父节点为 i/2（向下取整）。

    二叉树与度为 2 的树的区别：二叉树是有序树，左右子节点区分明确，即使只有一个子节点也要指明是左还是右；而度为 2 的树不要求区分左右次序。

    2. 二叉树的遍历

    遍历是二叉树最基本的操作，分为深度优先遍历和广度优先遍历两大类。

    深度优先遍历（DFS）——递归或栈实现：

    遍历方式 遍历顺序 特点 典型应用
    前序遍历 根 → 左 → 右 先访问根节点，再递归遍历左右子树 树的序列化、表达式树求值
    中序遍历 左 → 根 → 右 在 BST 中输出递增序列 BST 排序输出、判断是否为 BST
    后序遍历 左 → 右 → 根 先处理子节点再处理父节点 树的删除、计算目录大小

    广度优先遍历（BFS）——队列实现：

    • 层序遍历：从上到下、从左到右逐层访问每个节点。
    • 每轮迭代记录当前层的节点数量，可将各层节点区分输出（如按层打印二叉树）。
    • 常用于求树的高度、宽度、判断是否为完全二叉树。

    遍历序列的互推关系：

    • 前序遍历的第一个节点为根节点。
    • 中序遍历中，根节点左侧为左子树的中序序列，右侧为右子树的中序序列。
    • 后序遍历的最后一个节点为根节点。
    • 已知前序 + 中序可唯一确定一棵二叉树；已知后序 + 中序也可唯一确定；但仅知前序 + 后序不能唯一确定。

    3. 二叉搜索树（BST）

    定义与性质：

    • 左子树中所有节点的值均小于根节点的值。
    • 右子树中所有节点的值均大于根节点的值。
    • 左右子树本身也分别为二叉搜索树。
    • 对 BST 进行中序遍历，得到的结果是一个严格递增的序列。

    核心操作：

    • 查找：从根节点出发，若目标值等于当前节点值则返回；若目标值小于当前节点值则进入左子树继续查找；否则进入右子树。每轮比较排除一半子树，平均时间复杂度 O(log n)。
    • 插入：先执行一次查找操作以定位插入位置，查找未命中时，在查找路径的最后空位插入新节点。
    • 删除：分三种情况处理——
      • 被删除节点为叶子节点（度为 0）：直接删除，将父节点的相应指针置空。
      • 被删除节点只有一个子节点（度为 1）：用该子节点替换被删除节点。
      • 被删除节点有两个子节点（度为 2）：用右子树的最小节点（或左子树的最大节点）替换被删除节点的值，然后删除该替换节点（右子树最小节点一定是叶子或只有右子树的节点，其删除可归为前两种情况）。

    BST 的退化问题：如果插入的数据是有序的（递增或递减），BST 会退化为一条单链表，所有操作退化为 O(n)。为解决这一问题，引入了平衡二叉树。

    判定 BST 的方法：

    • 方法一：中序遍历是否严格递增（可以通过递归遍历时记录前驱节点值进行比较）。
    • 方法二：递归判断每个节点是否在允许的值范围内（以 min 和 max 参数约束左右子树的范围）。

    4. 平衡二叉树（AVL 树）

    AVL 树是严格平衡的二叉搜索树，任一节点的左右子树高度差不超过 1。

    平衡因子：左子树高度 − 右子树高度。平衡因子取值只能为 -1、0 或 1，否则需要旋转调整。

    四种旋转操作（在不破坏 BST 性质的前提下恢复平衡）：

    失衡情形 平衡因子特征 旋转方式 说明
    LL（左左） 节点平衡因子 >1，左子节点平衡因子 ≥0 右旋 在左子树的左子树插入了新节点
    RR（右右） 节点平衡因子 <−1，右子节点平衡因子 ≤0 左旋 在右子树的右子树插入了新节点
    LR（左右） 节点平衡因子 >1，左子节点平衡因子 <0 先左旋左子节点，再右旋当前节点 在左子树的右子树插入了新节点
    RL（右左） 节点平衡因子 <−1，右子节点平衡因子 >0 先右旋右子节点，再左旋当前节点 在右子树的左子树插入了新节点

    旋转操作的核心：旋转后子树的中序遍历序列不变（保持 BST 性质），且使整棵子树恢复高度平衡。

    AVL 树的性能：

    • 查找、插入、删除的时间复杂度均为 O(log n)。
    • 插入和删除后需要沿着节点到根节点的路径自底向上检查平衡因子，必要时进行旋转操作。
    • AVL 树的平衡要求比红黑树更严格，因此旋转操作更频繁，但查找效率更高。

    5. 堆（Heap）

    堆是一种特殊的完全二叉树，用数组存储。堆中的每个节点都满足特定的序关系。

    两种基本类型：

    • 大根堆（大顶堆）：每个节点的值大于等于其子节点的值，堆顶是最大值。
    • 小根堆（小顶堆）：每个节点的值小于等于其子节点的值，堆顶是最小值。

    数组表示（通常从下标 1 开始）：

    • 父节点索引 parent = k / 2
    • 左子节点索引 left = 2k
    • 右子节点索引 right = 2k + 1

    核心操作：

    • 上浮（swim / shift-up）：插入新元素时先放至数组末尾，然后不断与其父节点比较，若不满足堆序则交换，直到堆序恢复或到达堆顶。时间复杂度 O(log n)。
    • 下沉（sink / shift-down）：删除堆顶后，将数组末尾元素移至堆顶，然后不断与其子节点中较大（大根堆）或较小（小根堆）者交换，直到堆序恢复。时间复杂度 O(log n)。
    • 建堆：从最后一个非叶子节点（索引 n/2）开始，逐一执行下沉操作，直到根节点。时间复杂度为 O(n)——这一结果符合直觉：大部分节点位于树的底层，从底层向上调整成本更低。

    典型应用：

    • 优先队列的底层实现。
    • 堆排序：建堆 O(n) + n 次取堆顶并下沉 O(log n) = O(n log n)，空间复杂度 O(1)。
    • Top K 问题：用大小为 K 的小根堆遍历数据，堆中始终保留当前最大的 K 个元素。
    • 求中位数：用一个大根堆存储较小的一半数据，一个小根堆存储较大的一半数据。

    6. 红黑树

    红黑树是一种弱平衡二叉搜索树，通过节点颜色和约束条件保证树的大致平衡，从而使操作时间复杂度保持在 O(log n)。

    五大性质：

    1. 每个节点不是红色就是黑色。
    2. 根节点为黑色。
    3. 叶子节点（NIL 空节点）为黑色。
    4. 红色节点的子节点必须是黑色（即不能出现连续的红色节点）。
    5. 从任意节点到其每个叶子节点的所有路径上包含相同数量的黑色节点（黑色平衡）。

    核心操作：

    • 插入：新插入的节点默认为红色。插入后通过变色、左旋、右旋等操作恢复红黑性质。最多需要两次旋转。
    • 删除：较为复杂，可能需要多次旋转和变色来恢复红黑性质。

    实际应用：

    • Java 的 TreeMap 和 TreeSet 基于红黑树实现。
    • Java 8 的 HashMap 在链表长度 ≥8 且数组长度 ≥64 时，将链表转换为红黑树来优化冲突时的查找效率。
    • Linux 内核中的完全公平调度器（CFS）和 epoll 事件驱动机制使用了红黑树。
    • C++ STL 中的 std::map 和 std::set 通常也基于红黑树实现。

    AVL 树与红黑树的对比：

    对比维度 AVL 树 红黑树
    平衡程度 严格平衡（高度差 ≤1） 弱平衡（最长路径不超过最短路径的 2 倍）
    查找效率 更高（更矮） 略低
    插入/删除效率 较低（频繁旋转） 较高（旋转次数少）
    每节点额外空间 平衡因子（通常存储高度） 颜色标记（1 比特）
    适用场景 查找密集型 插入删除密集型

    7. 哈夫曼树与哈夫曼编码

    哈夫曼树（Huffman Tree）：带权路径长度（WPL, Weighted Path Length）最小的二叉树，也称最优二叉树。

    构造方法：

    1. 将所有节点视为独立的树（森林），每个节点对应一个权值。
    2. 每次从森林中选取权值最小的两棵树合并为一棵新树，新树的根节点权值为两子树权值之和。
    3. 重复步骤 2 直到森林中只剩一棵树，即为哈夫曼树。

    带权路径长度（WPL）：所有叶子节点的权值乘以从根节点到该叶子节点的路径长度之和。哈夫曼树的 WPL 最小。

    哈夫曼编码：

    • 将哈夫曼树中从根到每个叶子节点的路径编码为二进制序列：向左走编码为 0，向右走编码为 1。
    • 哈夫曼编码是前缀码（prefix code），即没有任何一个编码是另一个编码的前缀，因此可以无歧义地解码。
    • 出现频率越高的字符，对应的叶子节点越靠近根，编码越短，从而压缩数据。
    • 应用场景：数据压缩（JPEG、ZIP 等文件格式中的熵编码步骤）。

    三、图

    图由顶点（Vertex）和边（Edge）组成，记为 G = (V, E)，用于表示任意多对多的关系。

    1. 图的分类与术语

    按方向分类：

    • 无向图：边没有方向，(u, v) 表示 u 和 v 之间的双向关系。好友关系图、交通路线图为典型应用。
    • 有向图：边有方向，<u, v> 表示从 u 到 v 的单向关系。关注关系、网页链接图为典型应用。

    按连通性分类：

    • 连通图：无向图中任意两个顶点之间都存在路径。有向图中对应的概念为强连通图。
    • 非连通图：存在至少一对顶点之间没有路径。

    按边权重分类：

    • 有权图：边带有权重值，表示距离、成本、容量等信息。
    • 无权图：边不带权重，或认为所有边权重相同。

    图的常用术语：

    • 邻接：两个顶点之间存在边时称它们相互邻接。
    • 路径：从顶点 A 到 B 经过的边构成的序列。
    • 度：与顶点相关联的边数。在有向图中分为入度（指向该顶点的边数）和出度（从该顶点出发的边数）。
    • 环/回路：起点和终点相同的路径。

    2. 图的存储结构

    存储方式 实现方法 空间复杂度 特点
    邻接矩阵 n × n 二维数组，元素表示边的存在与否 O(V²) 判断任意两点是否相邻 O(1)，适合稠密图；无向图的矩阵对称
    邻接表 每个顶点维护一个链表（或动态数组），存储其所有邻接顶点 O(V+E) 遍历邻接点快，适合稀疏图；可进一步用哈希表或红黑树优化

    邻接矩阵的优缺点：优点是判断边是否存在非常快速（直接读取矩阵元素），适合矩阵运算（如 Floyd 算法）；缺点是空间占用与顶点数的平方成正比，当图稀疏时浪费严重。

    邻接表的优缺点：优点是空间效率高（只存储实际存在的边），遍历邻接点方便；缺点是需要遍历链表来判断两点之间是否有边。

    3. 图的遍历

    深度优先搜索（DFS）：

    • 从一个顶点出发，递归地访问其未访问的邻接顶点，直到无法继续，然后回溯。
    • 实现方式：递归或显式栈。
    • 应用：连通分量检测、拓扑排序、路径搜索、图的着色。

    广度优先搜索（BFS）：

    • 从一个顶点出发，先访问其所有邻接顶点，再逐层向外扩展。
    • 实现方式：队列。
    • 应用：无权图的最短路径、拓扑排序、连通分量检测。

    两者的对比：

    对比维度 DFS BFS
    依赖数据结构 栈（递归或显式） 队列
    遍历策略 沿一条路径深入到底再回溯 逐层向外扩张
    适用场景 路径搜索、连通性判断 最短路径（无权图）、拓扑排序
    空间复杂度 与图深度相关 与图宽度相关

    遍历序列的非唯一性：DFS 和 BFS 的遍历序列都不唯一，取决于顶点被访问的先后次序。

    4. 最小生成树

    在一个连通无向带权图中，最小生成树（MST, Minimum Spanning Tree）是包含所有顶点且边权值之和最小的树形子图。

    Prim 算法：

    • 从一个初始顶点开始，每次选取与当前已选顶点集距离最近的顶点加入，直到覆盖所有顶点。
    • 适合稠密图。
    • 使用优先队列优化时时间复杂度为 O(E log V)，不使用时为 O(V²)。

    Kruskal 算法：

    • 将所有边按权值从小到大排序，依次选取不构成环的边，直到选取了 V-1 条边。
    • 用并查集判断是否构成环。
    • 适合稀疏图。
    • 时间复杂度 O(E log E)。

    两者的区别：Prim 从顶点出发"生长"出最小生成树，Kruskal 从边出发"筛选"出最小生成树。

    5. 最短路径

    Dijkstra 算法（单源最短路径，要求边权非负）：

    • 维护一个 dist 数组记录源点到各顶点的当前最短距离。
    • 每次从未确定最短路径的顶点中选取 dist 值最小的顶点，标记为已确定，并松弛其所有邻接边（更新邻接顶点的 dist 值）。
    • 使用优先队列优化时时间复杂度为 O(E log V)。
    • Dijkstra 无法处理负权边，因为一旦标记为已确定，后续发现更短路径时无法修正。

    Bellman-Ford 算法（单源最短路径，可处理负权）：

    • 对 V-1 轮迭代，每轮尝试松弛所有边。
    • 第 V 轮如果仍有边可松弛，则存在负权环。
    • 时间复杂度 O(VE)。
    • 适用场景：图中可能存在负权边，或需要检测负权环。

    Floyd 算法（多源最短路径）：

    • 动态规划思想，递推公式：dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])。
    • 时间复杂度 O(V³)，空间复杂度 O(V²)。
    • 可同时处理所有顶点对之间的最短路径。

    6. 拓扑排序

    拓扑排序是对有向无环图（DAG, Directed Acyclic Graph）的所有顶点进行线性排序，使得对每条有向边 (u, v)，在排序中 u 都出现在 v 之前。

    Kahn 算法：

    1. 计算所有顶点的入度。
    2. 将所有入度为 0 的顶点入队。
    3. 出队一个顶点，加入拓扑序列，将其所有邻接顶点的入度减 1。若某个邻接顶点入度变为 0，则入队。
    4. 重复步骤 3 直到队列为空。
    5. 若拓扑序列中的顶点数小于总顶点数，说明图中存在环。

    应用场景：课程安排（先修课程关系）、编译依赖解析（Makefile、项目构建工具）、任务调度中的依赖排序。

    7. 并查集（Union-Find）

    并查集用于解决动态连通性问题，主要支持两种操作：连接两个元素（union）和判断两个元素是否属于同一集合（find）。

    四种实现及其性能：

    实现方式 find 复杂度 union 复杂度 特点
    Quick Find O(1) O(n) 同一集合所有元素 id 相同，find 快速但 union 需遍历整个数组
    Quick Union 树高（最坏 O(n)） 树高（最坏 O(n)） 用树结构表示集合，union 只需修改一个父指针，但树可能退化
    加权 Quick Union O(log n) O(log n) union 时将小树挂到大树根下，保证树高不超过 log n
    路径压缩 + 加权 近乎 O(1)（反阿克曼函数 α(n)） 近乎 O(1) 在 find 时将路径上所有节点直接指向根，进一步压缩树高

    路径压缩的原理：在执行 find 操作的同时，将沿途所有节点的父指针直接指向集合的根节点，使树结构更加扁平化。后续的 find 操作将更加高效。

    应用场景：Kruskal 算法判环、社交网络中的好友关系检测、路径查找、集合划分问题。

    四、排序

    排序是将一组数据按特定顺序重新排列的操作，是笔试中最常考查的算法知识领域。

    1. 排序算法总对比

    算法 平均时间复杂度 最坏时间复杂度 最好时间复杂度 空间复杂度 是否稳定 是否原地排序

    选择排序 O(n²) O(n²) O(n²) O(1) 否 是
    插入排序 O(n²) O(n²) O(n) O(1) 是 是
    希尔排序 O(n^1.3) 平均 O(n²) O(n) O(1) 否 是
    归并排序 O(n log n) O(n log n) O(n log n) O(n) 是 否
    快速排序 O(n log n) O(n²) O(n log n) O(log n) 否 是
    堆排序 O(n log n) O(n log n) O(n log n) O(1) 否 是
    计数排序 O(n + k) O(n + k) O(n + k) O(k) 是 否
    桶排序 O(n + k) O(n²) O(n + k) O(n + k) 是 否

    2. 各排序算法说明

    冒泡排序（Bubble Sort）：

    • 通过相邻元素的比较和交换，将最大（或最小）元素逐步"冒泡"到数组末尾。
    • 每趟排序确定一个元素的最终位置，共需 n-1 趟。
    • 可通过设置标志位优化：若一趟排序中没有发生交换，说明数组已有序，可提前终止。

    选择排序（Selection Sort）：

    • 每趟从待排序区间中选出最小（或最大）元素，将其放在已排序区间的末尾。
    • 与输入数据的初始顺序无关，无论数据是否有序都需要执行完整的比较。
    • 交换次数少（最多 n-1 次），但比较次数固定为 O(n²)。

    插入排序（Insertion Sort）：

    • 将数组分为已排序区和未排序区，每次从未排序区取出第一个元素，插入到已排序区的适当位置。
    • 当输入数据基本有序时效率极高，时间复杂度接近 O(n)。
    • 插入过程需要移动元素，平均时间复杂度 O(n²)。

    希尔排序（Shell Sort）：

    • 插入排序的改进版。先将数组按一定增量分组，对每组进行插入排序，然后逐步缩小增量，增量为 1 时完成最终排序。
    • 突破了 O(n²) 的复杂度界限，但仍然是 O(n²) 级别的退化。
    • 增量序列的选择影响性能（如 Hibbard 增量、Sedgewick 增量等）。

    归并排序（Merge Sort）：

    • 分治策略：将数组递归地一分为二，对两个子数组分别排序，然后合并两个有序子数组。
    • 合并操作需要借助临时数组，空间复杂度 O(n)。
    • 自顶向下（递归拆分）和自底向上（迭代合并）两种实现方式。
    • 外部排序的基础——处理大规模数据时需要将数据分块读入内存，逐块排序后再多路归并。

    快速排序（Quick Sort）：

    • 分治策略：选择 pivot（基准元素），通过 partition（划分）操作将数组分为小于 pivot 和大于 pivot 两部分，然后递归排序两部分。
    • Partition 的核心过程：从两端向中间扫描，将大于 pivot 的元素换到右边，小于 pivot 的换到左边。
    • 实际应用中最快的通用排序算法，内循环指令少且能充分利用 CPU 缓存。
    • 最坏情况发生在每次划分极不均匀时（如已有序数组且 pivot 选择不当），退化为 O(n²)。
    • 常用优化：随机选择 pivot、三数取中、小数组改用插入排序。

    堆排序（Heap Sort）：

    • 利用堆这种数据结构进行排序，分为建堆和排序两个阶段。
    • 建堆：将无序数组构建为大根堆（或小根堆），时间复杂度 O(n)。
    • 排序：反复将堆顶元素与末尾元素交换，并下沉恢复堆序，共执行 n-1 次。
    • 无法利用 CPU 缓存的空間局部性，因为比较和交换的元素通常不相邻。

    计数排序（Counting Sort）：

    • 非比较排序，适用于数据范围有限的整数排序。
    • 统计每个值出现的次数，然后根据计数信息将元素放到正确位置。
    • 时间复杂度 O(n + k)，其中 k 是值的范围。当 k 较小时效率极高。

    桶排序（Bucket Sort）：

    • 将数据分到有限数量的桶中，每个桶内部独立排序（可用其他排序算法），然后按桶的顺序依次输出。
    • 桶内元素分布均匀时效率高，分布不均匀时可能退化。

    基数排序（Radix Sort）：

    • 按位排序，从最低有效位到最高有效位依次进行排序。
    • 每位内部可以使用计数排序等稳定排序。
    • 适用于定长整数或短字符串的排序。

    3. 排序选择指导

    • 数据量较小（n < 50）：插入排序。
    • 数据量较大且需要稳定排序：归并排序。
    • 数据量较大且对稳定性无要求：快速排序（一般场景）或堆排序（空间敏感场景）。
    • 数据范围很小且为整数：计数排序。
    • 数据已基本有序：插入排序（接近 O(n)）。
    • 需要外部排序（数据量大到无法全部装入内存）：归并排序的多路归并版本。

    五、查找

    1. 顺序查找

    从线性表的一端开始逐个比较，直到找到目标或遍历完整个表。

    • 时间复杂度：O(n)。
    • 对数据的有序性没有要求，无序数据只能使用顺序查找。
    • 是查找算法中最基础、最普适的方法。

    2. 二分查找

    二分查找（Binary Search）是在有序数组中高效查找目标值的方法。

    基本原理：每次将搜索范围缩小一半。比较中间元素与目标值，若相等则返回；若中间元素小于目标值，则在右半部分继续查找；否则在左半部分继续查找。

    基本模板的三要素：

    • 左右边界初始化：l = 0, h = n-1（或 h = n，取决于边界定义）。
    • 循环条件：l <= h（左右闭区间）或 l < h（左闭右开区间）。
    • 中间值计算：m = l + (h - l) / 2（防止 l + h 溢出）。

    主要变体：

    • 查找第一个等于目标值的位置（左边界二分）：找到目标值后不立即返回，继续向左区间收缩。
    • 查找最后一个等于目标值的位置（右边界二分）：找到目标值后不立即返回，继续向右区间收缩。
    • 查找第一个不小于目标值的位置（lower_bound）：条件从 a[m] == target 变为 a[m] >= target。
    • 查找最后一个不大于目标值的位置（upper_bound − 1）：条件从 a[m] == target 变为 a[m] <= target。
    • 旋转数组中的查找：先判断 mid 位于旋转点的哪一侧（左半有序还是右半有序），再判断 target 是否在有序区间内。

    二分查找的适用条件：数据必须有序且支持随机访问（即数组结构）。链表中无法使用二分查找。

    六、哈希表

    哈希表（Hash Table）是一种根据键直接访问数据存储位置的数据结构，通过哈希函数将键映射为数组下标。

    1. 哈希函数

    哈希函数将任意大小的键转换为固定范围的整数值（哈希值）。

    设计原则：

    • 一致性：相等的键必须产生相等的哈希值。
    • 高效性：计算过程简单快速。
    • 均匀性：哈希值应尽可能均匀分布在地址空间中，减少冲突。

    常用方法——除留余数法：hash(k) = k % M，其中 M 通常取素数。素数 M 能减少哈希碰撞的概率，因为如果 M 是合数，键的分布模式可能被保留。

    Java 中 hashCode() 的设计：对于组合类型，如自定义类和字符串，通常使用 R 进制加权和的方式计算哈希值（如 hash = 31 * hash + field.hashCode()），其中乘数 31 是质数且有良好的乘法性能优化（31 * i = (i << 5) - i）。

    2. 冲突解决方法

    当两个不同的键映射到同一个哈希值时，称为哈希冲突。以下是主要的解决策略：

    方法 原理 优点 缺点
    链地址法（Separate Chaining） 每个槽位存一个链表（或树），冲突的元素链接在链表中 实现简单，不受冲突数量限制 链表过长时查找退化为 O(k)
    开放定址法——线性探测 冲突后依次检查下一个槽位（+1, +2, …） 空间利用率高，无额外指针开销 容易产生聚集现象，删除需要懒惰标记
    开放定址法——平方探测 冲突后按平方增量探测（+1², +2², …） 缓解线性探测的聚集问题 可能探测不到所有槽位
    再哈希法（Double Hashing） 冲突后用第二个哈希函数计算新地址 分布更均匀 计算开销大

    链地址法的具体实现：每个桶中存储一个链表（或红黑树，用于优化长链表的查找效率）。Java 8 的 HashMap 中，当链表长度 ≥8 且数组长度 ≥64 时，将链表转换为红黑树，使查找时间复杂度从 O(k) 降至 O(log k)。

    3. 负载因子与扩容

    负载因子（Load Factor）：α = 元素个数 / 桶数。负载因子衡量了哈希表的满度程度。

    • α 越大，冲突概率越高，查找性能越差。
    • α 越小，空间浪费越多。

    扩容机制：当负载因子超过阈值（如 Java HashMap 的 0.75）时，触发扩容。扩容通常将桶数组大小翻倍（或近似翻倍），然后将所有已存在的键值对重新哈希到新的数组中。扩容是哈希表性能的保证，但也带来了偶发的 O(n) 开销。

    Java HashMap 的核心实现要点：

    • 底层结构：数组 + 链表/红黑树。
    • 初始容量：16。
    • 负载因子阈值：0.75。
    • put 流程：计算 key 的哈希值 → 定位桶 → 若为空则直接放入；若非空则遍历链表/红黑树查找 key——若找到则更新 value，否则插入新节点。插入后检查是否需要扩容或将链表转为红黑树。
    • 哈希表的优势：理想情况下查找、插入、删除操作均为 O(1)。

    七、算法分析与复杂度

    1. 大 O 表示法

    大 O 表示法描述算法运行时间的渐进上界，它忽略常数因子和低阶项，只关注输入规模趋近无穷大时算法的增长趋势。例如，对于表达式 3n² + 5n + 10，大 O 表示为 O(n²)。

    增长数量级排序（从小到大）：

    O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

    复杂度 对 n=10⁶ 的直观感受 典型算法
    O(1) 常数时间 数组随机访问
    O(log n) 约 20 步 二分查找
    O(n) 约 1 秒 线性遍历
    O(n log n) 约 20 秒 归并排序、快速排序平均
    O(n²) 约 11.5 天 选择排序、冒泡排序
    O(2ⁿ) 不可计算 穷举搜索

    2. 递归复杂度分析——主定理

    对于形如 T(n) = aT(n/b) + f(n) 的递归式（其中 a ≥ 1, b > 1，f(n) 是渐进正的函数），主定理提供了三种情况的直接结论：

    • 情况一：若 f(n) 的增长量级小于 n^(log_b a)，则 T(n) = Θ(n^(log_b a))。
    • 情况二：若 f(n) 的增长量级等于 n^(log_b a)，则 T(n) = Θ(n^(log_b a) · log n)。
    • 情况三：若 f(n) 的增长量级大于 n^(log_b a) 且满足正则条件 af(n/b) ≤ cf(n)（c < 1），则 T(n) = Θ(f(n))。

    主定理的典型应用：二分查找 T(n) = T(n/2) + O(1) → 情况一 → O(log n)；归并排序 T(n) = 2T(n/2) + O(n) → 情况二 → O(n log n)。

    3. 均摊分析

    均摊分析用于评估一个操作序列的平均代价，而不是单个操作的最坏情况代价。

    典型例子——动态数组扩容：向动态数组中插入元素，绝大多数插入操作是 O(1) 的，只有当数组满时才触发 O(n) 的扩容操作。通过均摊分析：N 次插入操作的总代价 = N 次普通插入 + 若干次扩容的数据拷贝 = 约 2N 次操作，均摊到每次插入为 O(1)。

    八、贪心与动态规划

    1. 贪心算法

    贪心算法在每一步都做出当前看起来最优的选择，期望通过一系列局部最优选择来达到全局最优。

    适用条件：

    • 最优子结构：问题的最优解包含其子问题的最优解。
    • 贪心选择性质：问题的全局最优解可以通过一系列局部最优选择得到（即一旦做出选择就不能回退）。

    经典问题：

    • 活动选择（按结束时间最早排序，每次选结束时间最早的相容活动）。
    • 零钱找零（在特定硬币面额体系下，优先使用大面额硬币）。
    • 区间调度（求最多的不相交区间数）。
    • 哈夫曼编码（每次合并权值最小的两个节点）。

    2. 动态规划

    动态规划（DP, Dynamic Programming）通过将复杂问题分解为重叠子问题，并记录子问题的解来避免重复计算。

    核心要素：

    • 最优子结构：问题的最优解包含子问题的最优解。
    • 重叠子问题：不同子问题会反复被求解（与分治法的关键区别——分治法的子问题通常是独立的）。
    • 状态定义：如何表示一个子问题的解（如一维数组 dp[i] 或二维数组 dp[i][j]）。
    • 状态转移方程：描述状态之间的递推关系。
    • 备忘录或 DP 表：存储已求解的子问题结果，避免重复计算。

    贪心与动态规划的区别：

    对比维度 贪心算法 动态规划
    决策方式 一次选择，不回退 多阶段决策，保留所有可能
    依赖关系 只依赖当前状态 依赖多个子问题
    适用性 要求贪心选择性质 要求最优子结构
    典型问题 活动选择、Huffman 编码 背包问题、LCS、LIS

    经典动态规划问题：

    问题 状态定义 核心思想
    斐波那契数列 dp[i] = 第 i 个数 dp[i] = dp[i-1] + dp[i-2]，最简单的 DP 入门
    01 背包 dp[i][w] = 前 i 件物品在容量 w 下的最大价值 dp[i][w] = max(dp[i-1][w], dp[i-1][w-wi] + vi)，可选优化为滚动数组一维
    最长公共子序列（LCS） dp[i][j] = s1[0..i] 和 s2[0..j] 的 LCS 长度 若字符相等则 +1，否则取左/上的较大值
    最长递增子序列（LIS） dp[i] = 以 a[i] 结尾的最长递增子序列长度 dp[i] = max(dp[j] + 1) for j < i and a[j] < a[i]，可使用二分优化到 O(n log n)
