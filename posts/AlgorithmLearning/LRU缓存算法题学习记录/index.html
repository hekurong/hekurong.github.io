    LRU缓存算法题学习记录

    2025-09-20 | in Algorithm Learning
    LRU

    LRU 缓存到底要做什么？

    在动手写代码之前，最重要的一步是明确我们的目标。LRU 缓存机制的核心是：当缓存满了以后，优先淘汰最久没有被使用过的数据。

    我们来分解一下 LRUCache 需要实现的功能：

    1. Constructor(capacity int): 初始化一个固定容量的缓存。
    2. Get(key int):
      • 如果 key 存在，返回对应的 value。
      • 关键点：这次访问使得该 key 成为了“最近使用的”，所以需要更新它的位置。
      • 如果 key 不存在，返回 -1。
    3. Put(key int, value int):
      • 如果 key 存在，更新 value。
      • 关键点：这次更新也使得该 key 成为了“最近使用的”，需要更新它的位置。
      • 如果 key 不存在，插入这个新的键值对。
      • 关键点：插入后，如果缓存的尺寸超过了 capacity，则必须淘汰“最久未使用的”那个元素。

    总结一下，我们需要一个数据结构，它必须同时满足以下几个要求：

    • 快速查找：Get 和 Put 都需要快速地判断一个 key 是否存在。
    • 有序存储：需要一种机制来记录所有 key 的“新旧”顺序，方便我们找到“最久未使用的”那个。
    • 快速增删：当一个 key 被访问（Get 或 Put）时，我们要能快速地将它移动到“最新”的位置。当缓存满了，也要能快速地删除“最旧”的那个。

    哈希表 + 双向链表

    这是本题的核心套路。没有任何单一的基础数据结构能同时满足上述所有要求，所以需要组合。

    • 为什么需要哈希表 (map)？
      • 为了实现 O(1) 复杂度的快速查找。我们可以用 map[int]*Node 来存储 key 到链表节点的映射。
    • 为什么需要链表？
      • 为了维护元素的“新旧”顺序。链表天然有序。我们可以规定，越靠近链表头部的元素，就是“最近使用”的；越靠近链表尾部的，就是“最久未使用”的。
    • 为什么必须是双向链表，而不是单向？
      • 因为我们需要 O(1) 复杂度的删除操作。想象一下，当我们要删除一个节点（比如缓存满了要淘汰尾部节点，或者 Get 了一个中间节点要把它移动到头部），我们需要修改这个节点前面那个节点的 Next 指针。在单向链表中，找到前驱节点需要从头遍历，时间复杂度是 O(n)。而双向链表每个节点都存了 Pre 和 Next 指针，使得我们可以在 O(1) 的时间内完成任意节点的删除。

    结论：哈希表 (map) + 双向链表 (Doubly Linked List) 是解决此问题的完美组合。

    • 哈希表 负责快速定位 (Get）：map[key] -> Node。
    • 双向链表 负责维护顺序和快速增删 (Put/淘汰）：head <-> node1 <-> node2 <-> tail。

    代码实现

    1. 定义数据结构

     1// LRUCache holds the core logic for the cache.
 2type LRUCache struct {
 3    cache      map[int]*Node // Hash map for O(1) lookups: key -> Node pointer
 4    head, tail *Node          // Dummy head and tail nodes to handle edge cases
 5    cap        int           // The capacity of the cache
 6}
 7
 8// Node represents a node in the doubly linked list.
 9type Node struct {
10    Val, Key  int           // Store both key and value
11    Pre, Next *Node         // Pointers to the previous and next nodes
12}

    2. 初始化 Constructor

     1// Constructor initializes a new LRUCache with a given capacity.
 2func Constructor(capacity int) LRUCache {
 3    head, tail := &Node{}, &Node{}
 4    head.Next = tail
 5    tail.Pre = head
 6    return LRUCache{
 7       cache: make(map[int]*Node),
 8       head:  head,
 9       tail:  tail,
10       cap:   capacity,
11    }
12}

    3. 编写核心辅助函数

    为了让 Get 和 Put 的逻辑更清晰，我们把链表操作封装成独立的辅助函数。

    • removeNode(node *Node): 从链表中移除一个节点。
    • addToHead(node *Node): 将一个节点添加到链表头部（head 之后）。
    • moveToHead(node *Node): 组合技，先移除再添加到头部，用于更新节点为“最新”。
     1// removeNode detaches a node from the doubly linked list.
 2func (this *LRUCache) removeNode(node *Node) {
 3    node.Pre.Next = node.Next
 4    node.Next.Pre = node.Pre
 5}
 6
 7// addToHead adds a node right after the dummy head.
 8func (this *LRUCache) addToHead(node *Node) {
 9    node.Pre = this.head
10    node.Next = this.head.Next
11    this.head.Next.Pre = node
12    this.head.Next = node
13}
14
15// moveToHead moves an existing node to the head of the list.
16func (this *LRUCache) moveToHead(node *Node) {
17    this.removeNode(node)
18    this.addToHead(node)
19}

    4. 实现 Get 和 Put

    有了辅助函数，Get 和 Put 的逻辑就变得非常直观了。

    Get(key int) 逻辑:

    1. 用哈希表查找 key。
    2. 如果不存在，返回 -1。
    3. 如果存在，我们拿到了节点 node。这个节点刚刚被访问，所以要把它更新为“最新”。直接调用 moveToHead(node)。
    4. 返回 node.Val。
    1func (this *LRUCache) Get(key int) int {
2    if node, ok := this.cache[key]; ok {
3       // The node was accessed, move it to the head to mark it as recently used.
4       this.moveToHead(node)
5       return node.Val
6    }
7    // Key not found.
8    return -1
9}

    Put(key int, value int) 逻辑:

    1. 先用哈希表查找 key，看是“更新”还是“插入”。
    2. 情况一：Key 已存在。
      • 更新 node 的 Val。
      • 因为它被访问了，调用 moveToHead(node) 将其更新为最新。
    3. 情况二：Key 不存在。
      • 检查容量：if len(this.cache) == this.cap，如果缓存满了：
        • 找到最久未使用的节点，也就是哨兵 tail 的前一个节点 tail.Pre。
        • 从链表中删除它：removeNode(tail.Pre)。
        • 从哈希表中也删除它：delete(this.cache, tail.Pre.Key)。
      • 创建一个新的 Node。
      • 将新节点添加到链表头部：addToHead(newNode)。
      • 将新节点存入哈希表：this.cache[key] = newNode。
     1func (this *LRUCache) Put(key int, value int) {
 2    // Case 1: The key already exists.
 3    if node, ok := this.cache[key]; ok {
 4       node.Val = value
 5       this.moveToHead(node) // Mark it as recently used.
 6       return
 7    }
 8
 9    // Case 2: The key does not exist.
10    // Check if the cache is full.
11    if len(this.cache) == this.cap {
12       // Evict the least recently used element, which is the one before the tail.
13       tail := this.tail.Pre
14       this.removeNode(tail)
15       delete(this.cache, tail.Key)
16    }
17
18    // Add the new node.
19    newNode := &Node{Key: key, Val: value}
20    this.addToHead(newNode)
21    this.cache[key] = newNode
22}

    LRU + 过期时间 (TTL)

    需求变更分析

    1. Constructor(capacity int, ttl int): 构造函数现在需要接收一个 ttl 参数，单位通常是秒。
    2. Put(key, value):
      • 当插入一个新的键值对时，需要记录它的过期时间点，即 当前时间 + ttl。
      • 当更新一个已存在的键值对时，同样需要重置它的过期时间点。
    3. Get(key):
      • 在返回一个值之前，必须检查它是否已经过期。
      • 如果已过期，应将其从缓存中删除，并返回 -1 (表现得好像它不存在一样)。
      • 如果未过期，才执行标准的 LRU 逻辑：把它移动到链表头部并返回值。

    核心设计决策：如何处理过期数据？

    这里有两种主流策略：

    1. 被动删除 (Lazy Eviction):
      • 我们不主动去监控哪些 key 过期了。
      • 只在 Get 一个 key 的时候，才去检查它的时间戳。如果发现它过期了，就地删除它。
      • 优点：实现简单，没有额外的性能开销。
      • 缺点：如果一个过期的 key 再也不被访问，它会一直占据着内存，直到被 LRU 规则淘汰出去。
    2. 主动删除 (Eager Eviction):
      • 可以通过一个后台的定时任务，周期性地扫描整个缓存，删除所有过期的 key。
      • 或者，使用一个额外的数据结构（如最小堆或另一个按过期时间排序的链表）来专门管理过期，这样可以快速找到下一个要过期的元素。
      • 优点：可以更及时地释放内存。
      • 缺点：实现复杂，会引入额外的复杂度和性能开销（例如，后台扫描可能导致锁竞争）。

    面试中的选择：在面试的有限时间内，被动删除 (Lazy Eviction) 是最推荐的方案。它清晰地展示了你处理核心逻辑的能力，而不会陷入过于复杂的实现细节中。

    所以，我们的策略是：在 Get 时检查过期，并在 Put 时更新过期时间。

    代码实现

    1. 升级数据结构

    Node 结构体需要增加一个字段来存储它的“死亡时间”。

     1import "time"
 2
 3// Node now includes an expiration timestamp.
 4type Node struct {
 5    Val, Key  int
 6    Pre, Next *Node
 7    ExpiresAt int64 // Unix timestamp (seconds) when this entry expires.
 8}
 9
10// The main struct now includes the TTL duration.
11type LRUCacheWithTTL struct {
12    cache      map[int]*Node
13    head, tail *Node
14    cap        int
15    ttl        int64 // Time-to-live in seconds for each entry.
16}

    2. 修改 Constructor

     1// Constructor now accepts a TTL value.
 2func Constructor(capacity int, ttl int) LRUCacheWithTTL {
 3    head, tail := &Node{}, &Node{}
 4    head.Next = tail
 5    tail.Pre = head
 6    return LRUCacheWithTTL{
 7       cache: make(map[int]*Node),
 8       head:  head,
 9       tail:  tail,
10       cap:   capacity,
11       ttl:   int64(ttl),
12    }
13}

    3. 修改 Get 方法

    这是变化最大的地方。在找到节点后，我们必须先检查它的存活状态。

     1// Get must now check for expiration before returning a value.
 2func (this *LRUCacheWithTTL) Get(key int) int {
 3    if node, ok := this.cache[key]; ok {
 4       // Check if the node has expired.
 5       if time.Now().Unix() > node.ExpiresAt {
 6           // It's expired. Remove it from the cache.
 7           this.removeNode(node)
 8           delete(this.cache, key)
 9           return -1 // Act as if it doesn't exist.
10       }
11
12       // Not expired, proceed with standard LRU logic.
13       this.moveToHead(node)
14       return node.Val
15    }
16
17    return -1
18}

    4. 修改 Put 方法

    Put 的逻辑也需要更新，无论是插入还是更新，都要设置新的过期时间。

     1// Put must now set or reset the expiration time.
 2func (this *LRUCacheWithTTL) Put(key int, value int) {
 3    // Calculate the new expiration time.
 4    newExpiresAt := time.Now().Unix() + this.ttl
 5
 6    // Case 1: The key already exists.
 7    if node, ok := this.cache[key]; ok {
 8       node.Val = value
 9       node.ExpiresAt = newExpiresAt // Reset its expiration time.
10       this.moveToHead(node)
11       return
12    }
13
14    // Case 2: The key does not exist.
15    // Check for capacity and evict the LRU element if full.
16    if len(this.cache) == this.cap {
17       tail := this.tail.Pre
18       this.removeNode(tail)
19       delete(this.cache, tail.Key)
20    }
21
22    // Create and add the new node.
23    newNode := &Node{
24        Key:       key,
25        Val:       value,
26        ExpiresAt: newExpiresAt,
27    }
28    this.addToHead(newNode)
29    this.cache[key] = newNode
30}
